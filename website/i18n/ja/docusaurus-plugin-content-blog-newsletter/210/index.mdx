## ⚛️ React {#react}

[![React 19 RC1](/emails/issues/210/react19.jpg)](https://github.com/facebook/react/issues/29898#issuecomment-2477449973)

[**React 19 RC1 - 兄弟コンポーネントの事前ウォーミング**](https://github.com/facebook/react/issues/29898#issuecomment-2477449973)

以前、Suspenseの変更が原因でイベントハンドラ内でフェッチを開始しないアプリでウォーターフォールが発生する可能性があったため、React 19の初期リリースが中止されたことをお伝えしました。\
Reactチームは、早期にフェッチを開始するアプリのパフォーマンスを低下させることなく、Suspenseの兄弟コンポーネント事前レンダリングを再導入するために尽力してきました。彼らの解決策は「[事前ウォーミング](https://react.dev/blog/2024/04/25/react-19-upgrade-guide#improvements-to-suspense)」として完成しました。この問題を提起したReact QueryのメンテナーであるDominikも[この修正に満足していることを確認しています](https://bsky.app/profile/tkdodo.eu/post/3layw6wk5bs2b)。\
いつものように、これらの新機能はNext.JS（React Canaryを使用）ですでに利用可能です。これにはサーバーコンポーネントとアクション、新しい`use`フック、フォーム処理の改善（`useFormStatus`と`useOptimistic`）、そして小さいながらも長年待ち望まれていた変更点——`forwardRef`が不要になったことも含まれています！

## 📱 React-Native {#react-native}

[![Expo Workflows](/emails/issues/210/workflows.jpg)](https://expo.dev/blog/expo-workflows-automate-your-release-process)

[**Expo Workflows**](https://expo.dev/blog/expo-workflows-automate-your-release-process)
Expoは「Workflows」を発表しました。これはReact NativeとExpoアプリのための新しいCI/CDツールで、事前パッケージ化されたジョブ（Maestroを使用したE2Eテストなど）、自動化されたワークフロー、最適なパフォーマンスのためのスマートマシン選択により、アプリリリースを簡素化し加速します。GitHub Actionsのようなより一般的なCI/CDソリューションの直接の代替ではありませんが、探求する価値のあるエキサイティングな改善を約束しています。

🎉 [Expo Launch Party](https://expo.dev/launch-party) はまだ開催中です。彼らの発表を見逃さないでください！

* 📣 [Expo - チャネル一時停止機能をEAS Updateに導入 : Webページまたはeas cli経由でチャネルを一時停止可能](https://expo.dev/changelog/2024/11-14-channel-pausing)
* 📜 [Expo RouterにおけるユニバーサルReact Server Components（ベータ版）](https://expo.dev/blog/universal-react-server-components-developer-preview): Expo Routerは初めてユニバーサルReact Server Componentsを導入し、ネイティブとWebアプリの両方でサーバーサイドレンダリングを可能にします。本番エクスポートが利用できないため本番環境にはまだ準備が整っていませんが、シークレット、データベース、ファイルシステムなどのリソースに直接サーバーサイドでアクセスできるため、開発を簡素化し、クライアント-サーバー間の往復をなくすことでパフォーマンスを向上させるエキサイティングな開発です。
* 📜 [Expo DOM Componentsの魔法](https://expo.dev/blog/the-magic-of-expo-dom-components): Expo DOM Componentsは、Web機能をReact Nativeアプリに統合することを容易にし、WebViewの速度と段階的なネイティブ移行を組み合わせます。WebViewのパフォーマンスと一部のExpo機能との互換性に制限がありますが、将来の更新ではネイティブ統合の強化とパフォーマンスの最適化を目指しています。
