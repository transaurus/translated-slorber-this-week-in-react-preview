**[Axiom - Vercelアプリ向け最高のロギングプラットフォーム](https://axiom.co/vercel)**

😴 AxiomのVercelプロジェクト向けゼロ設定オブザーバビリティがあれば、安心して眠れます。

* Axiomの事前構築済みダッシュボードで、すべてのVercelログとバイタルを一覧表示し、特定のプロジェクトやデプロイメントをドリルダウンし、ワンクリックで関数のパフォーマンスを把握できます。
* [next-axiom](https://github.com/axiomhq/next-axiom)を使用すると、特別な設定なしにNext.jsプロジェクトのクライアント、エッジ、サーバーサイドのどこからでもログやイベントを送信できます。
* コードから構造化ログを簡単に送信し、Vercelログと一緒に分析できます。

Axiomはイベントデータの100%を効率的にキャプチャするため、サンプリングや保持について心配する必要はなく、ユーザー体験を推測する必要もありません。

## 📱 React-Native {#react-native}

今週のReact NativeニュースはBenediktがまとめました。フィードバック、提案、共有したいニュースがあれば[Twitter](https://twitter.com/bndkt)でお知らせください。

[![Expo SDK 50](/emails/issues/171/expo50.jpg)](https://expo.dev/changelog/2024/01-18-sdk-50k)

**[Expo SDK 50](https://expo.dev/changelog/2024/01-18-sdk-50)**

Expo SDK 50ではExpo Dev Toolsプラグインが導入され、新しいSQLiteとCamera APIが追加されました。また、`@expo/fingerprint`が追加され、更新されたJSバンドルが特定のネイティブビルドと互換性があるかどうかを判断する新しい方法が提供されます。このリリースでは、Expoでの[Webpackサポートの終了](https://blog.expo.dev/webpack-support-in-expo-cli-is-now-deprecated-e9831d7eb631)も発表され、Metroが推奨されるようになりました。Expo 50について最も手軽に学ぶには、Simonが約15分で解説する[YouTube動画](https://www.youtube.com/watch?v=shaGaiwu4Fg)を視聴するのがおすすめです。

[![Expo Router v3](/emails/issues/171/router3.jpg)](https://expo.dev/changelog/2024/01-23-router-3)

**[Expo Router v3](https://expo.dev/changelog/2024/01-23-router-3)**

Expo SDK 50とともに、Expo Router v3がリリースされました。多くのパフォーマンスと安定性の改善が行われ、ドキュメントも充実しました。特に注目すべきは、APIルートを使用したユニバーサルサーバーエンドポイントの実験的サポートが追加されたことです。Expo Routerは今や、最初のユニバーサルなフルスタックReactフレームワークとなりました！
