## React

[Fresh 1.0](https://deno.com/blog/fresh-is-stable?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Deno公式サポートのPreactフレームワークがv1となり、本番環境対応と見なされました。FreshはRemixとAstroのミックスのような印象です：デフォルトでJSなし、プログレッシブエンハンスメント、アイランドアーキテクチャを採用したマルチページアプリです。ビルドステップがなく、TypeScript/Denoアプリを数秒でEdgeに直接デプロイできます。

[How We Built Hydrogen](https://shopify.engineering/how-we-built-hydrogen?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Shopifyのeコマースストア構築用ReactメタフレームワークであるHydrogenがv1.0となり、React Server Componentsを最初に採用したフレームワークの1つです。この記事はフレームワーク作成に関する興味深いフィードバックで、React Server Components、Vite、Tailwind、プリロード/ウォーターフォール、GraphQLフラグメントなど、時間をかけて行われた様々な選択について詳しく説明しています...

[Next.js 12.2](https://nextjs.org/blog/next-12-2?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Next.jsの最新バージョンです。主なテーマはEdgeと標準化されたWeb APIの段階的な採用です：ミドルウェアだけでなく、APIルートとSSRも対象です。画像に関する改善や、WebAssemblyでのSWCプラグインシステムも含まれています。

[素晴らしいHTMLメールのワークフロー](https://www.joshwcomeau.com/react/wonderful-emails-with-mjml-and-mdx/?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Josh Comeauが、MDXを使ってメールを作成し、MJML、React、Next.jsを活用して全てのメールクライアントで動作する最終的なHTML出力を生成し、ConvertKit経由で送信し、ウェブ上でも利用可能にする方法を解説しています。

[Remixにおけるデータフロー](https://remix.run/blog/remix-data-flow?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Reactの初期モデルは`ui = f(state)`と要約できます。しかし、このモデルはネットワーク同期（つまりAPI呼び出し）を十分に考慮していません。Jimは、RemixがReactモデルを取り入れつつネットワークを組み込むことで、ローカル状態管理の必要性を減らす方法を説明しています。

[Effectが必要ないかもしれない](https://beta.reactjs.org/learn/you-might-not-need-an-effect?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Dan Abramovによって書かれた新しいドキュメントページがベータサイトにマージされました。Effectは外部システムとの同期のために存在します。多くのアンチパターンが具体的な例と課題と共に紹介されています。経験豊富な開発者でも読む価値があり、[レンダリング中のsetStateの使用](https://twitter.com/sebastienlorber/status/1541793224561463297?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)のような驚くべき内容も含まれています。特に`useSyncExternalStore`の例が気に入りました。

[React 18でデータをロードする推奨方法は？](https://www.reddit.com/r/reactjs/comments/vi6q6f/what_is_the_recommended_way_to_load_data_for/?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Dan AbramovがRedditで、useEffectを使ったデータ取得が理想的でない理由を説明しています。ただし、UXに問題がないアプリケーションをすぐに書き直す必要はありません。これらの問題はReact 18で新たに発生したものではなく、より詳細に文書化されただけです。React-Nativeのデータ取得に関するコメントも参照してください。
