## ⚛️ React {#react}

**[Gatsby v5 Alpha](https://www.gatsbyjs.com/blog/gatsby-v5-alpha-is-here-and-its-pretty-exciting/)**

Gatsbyのv5がアルファ版でリリースされ、2つの主要な新機能が追加されました：

* React Server Componentsに基づく[部分的なハイドレーション](https://www.gatsbyjs.com/blog/a-look-at-partial-hydration-in-gatsby-5/)のサポートと、クライアントコンポーネントの先頭に`"client export"`ディレクティブを追加する機能
* Gatsby Slices API。これは、共有コンポーネント（レイアウト/ナビゲーションバー/フッター）を再レンダリングする必要がある場合に、ビルド時間を大幅に改善します

**[Reactをデフォルトで高速かつ真にリアクティブにする](https://legendapp.com/open-source/legend-state/)**

Reactの再レンダリングの問題（デフォルトでは最適化されていない）を説明するインタラクティブな記事。Legend-Stateライブラリ（Observableベース）を使用して、Reactにリアクティビティをもたらす方法を示しています（SolidJSに似ています）。コードはデフォルトで最適化され、不要な再レンダリングを回避します。

**[Ezno - 実験的コンパイラ](https://kaleidawave.github.io/posts/introducing-ezno/)**

TypeScriptアノテーションとJSXに対応した、JavaScript向けの新しい実験的型チェックコンパイラ。依存型を利用して、コンパイル時に高度な静的解析と事前評価を行います（[prepack](https://prepack.io/)を思い起こさせます）。目標は、Reactのように見えるアプリをリアクティブな形式でコンパイルし、VDOMをスキップすることのようです。技術的に高度な内容です。

**[(Lazy) Components Maps](https://julesblom.com/writing/lazy-component-maps)**

条件付きレンダリングを行う際にコード分割を活用する有用なテクニックを紹介します。特に大規模なenumやunion型（例: `<Icon name="my-icon-name"/>` コンポーネント）で有効です。

**[Whyframe](https://whyframe.dev/)**

フロントエンドコンポーネントのレンダリングをiframe内で隔離できるツールです。[Webpack、Next.js、Docusaurusに対応](https://twitter.com/bluwyoo/status/1576947013450539008)。コンポーネントライブラリのドキュメントサイトで、サイト全体のCSSが表示コンポーネントに影響を与えないようにするのに便利です。
