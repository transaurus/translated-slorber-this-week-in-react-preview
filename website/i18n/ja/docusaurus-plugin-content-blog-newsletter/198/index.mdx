**[インタラクティブチュートリアルでライブラリの採用を促進](https://tutorialkit.dev/?utm_source=twir\&utm_medium=primary)**

JavaScriptフレームワークやライブラリの採用を増やしたいですか？ **StackBlitzのTutorialKit**は、ユーザーを引き込み、素早く学べる完全にインタラクティブなコードチュートリアルを作成できるCLI（コマンドラインインターフェース）ジェネレーターです。

TutorialKitを使い始めるには、`npm create tutorial`コマンドを実行してチュートリアルサイト生成ウィザードを起動するだけです。スキャフォールドされたプロジェクトは、基礎となるチュートリアルロジックとブラウザ内コードエディターを処理するので、コンテンツに集中できます。

新しいフレームワークをリリースする場合でも、既存のライブラリを更新する場合でも、TutorialKitはユーザーが最高の学習体験を得られるようにします。あなたのライブラリが注目されないままにならないようにしましょう。[今すぐ始める！](https://tutorialkit.dev/?utm_source=twir\&utm_medium=link)

**[Rspack 1.0](https://rspack.dev/blog/announcing-1-0)**

Rspackは非常に高速なRust製バンドラーで、Webpackとほぼ100%互換性があり、プラグインやローダーのエコシステム全体をそのまま利用できます。Webpackを使用している多くのReactアプリケーションにとって、今日から高速なバンドルを実現する優れた選択肢となるでしょう。互換性のおかげで導入も簡単ですが、[今後のReact Server Componentsのファーストクラスサポート](https://rspack.dev/blog/announcing-1-0#react-server-components-support)により将来性も保証されています。これが私が[DocusaurusをRspackに移行している](https://x.com/sebastienlorber/status/1828696665135431879)理由です。[Rsdoctor](https://rsdoctor.dev/)もチェックすることをお勧めします。Webpackとも互換性のある優れたツールで、ビルドのボトルネックを教えてくれます。
