## ⚛️ React {#react}

[![Astro 5.0](/emails/issues/212/astro.jpg)](https://astro.build/blog/astro-5/)

**[Astro 5.0](https://astro.build/blog/astro-5/)**

Astroの新しいメジャーバージョンがリリースされました。技術的には純粋なReactフレームワークではありませんが、Reactコミュニティで広く利用されており、インタラクティブなReactコンポーネントを簡単に統合でき、MDXに依存しています。主な特徴は以下の通りです：

* すべてのコンテンツを1つの使いやすい型安全なデータストアに統合するContent Layer。CMS、API、その他のソースからのコンテンツを含みます。
* 初期ページ応答を可能な限り高速に提供しつつ、大部分が静的なレイアウトに動的コンテンツを織り交ぜる能力を維持するServer Islands。React Server ComponentsやNext.jsのPartial Pre-Renderingに対するよりシンプルで興味深い代替手段です。
* プリレンダリングの簡素化、「ハイブリッド」オプションの削除
* 型安全な環境変数
* 新しいEnvironment APIを備えたVite 6
* 実験的機能：画像のクロッピング、レスポンシブ画像レイアウト、SVGコンポーネント

[![Radon IDE 1.0](/emails/issues/212/radon.jpg)](https://ide.swmansion.com/)

**[Radon IDE 1.0](https://ide.swmansion.com/)**

Software Mansionは、VSCode拡張機能（Cursorもサポート）を作成し、エディターをReact NativeとExpoの本格的なIDEに変えました。現在は安定版として正式リリースされています。有料製品ですが、低価格と開発者体験（DX）の向上を考慮すると、同社の素晴らしいオープンソース活動もサポートするため、導入は必須と言えます。[Radon IDEローンチ動画](https://www.youtube.com/watch?v=07Un9EfE8D4)で機能の概要を確認できます：シミュレーターパネルと設定、デバッグ、クリックで検査、URL駆動ナビゲーション、Storybook統合など。
