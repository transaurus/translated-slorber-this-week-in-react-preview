## ⚛️ React {#react}

[![NetlifyがGatsby Inc.を買収、コンポーザブルWebアーキテクチャの採用を加速](/emails/issues/134/netlify-gatsby.jpg)](https://www.netlify.com/press/netlify-acquires-gatsby-inc-to-accelerate-adoption-of-composable-web-architectures/)

**[NetlifyがGatsby Inc.を買収、コンポーザブルWebアーキテクチャの採用を加速](https://www.netlify.com/press/netlify-acquires-gatsby-inc-to-accelerate-adoption-of-composable-web-architectures/)**

今週のビッグニュースです。Gatsby Inc.がNetlifyに買収されました。このフレームワークは、EleventyやSolidJSとともにNetlify傘下で進化を続けます。クラウド機能（[Valhalla Content Hub](https://www.gatsbyjs.com/products/valhalla-content-hub/)を含む）はNetlifyの提供サービスに統合されます。新しいスローガンは「コンポーザブルアーキテクチャ」です。Gatsbyは現在最も人気のあるReactフレームワークではありませんが、この買収は理にかなっていると思います。特にコンテンツハブと彼らが取り組んできたCMS統合のためです。Gatsby Cloudの提供は、Netlifyのオプション有料機能よりも採用が難しく、Gatsbyをフレームワークとして使用しないサイトではさらに顕著です。

[![Reactでステートマシンをグローバルにする](/emails/issues/134/xstate.jpg)](https://stately.ai/blog/making-state-machines-global-in-react)

**[Reactでステートマシンをグローバルにする](https://stately.ai/blog/making-state-machines-global-in-react)**

XState 3.1で登場する新しい`createActorContext` APIの紹介です。このAPIは、Reactコンテキストに関連付けられたグローバルなXStateステートマシンの作成を容易にします。セレクターシステムを使用することも、プロバイダーにカスタムステートマシンを提供することも可能です（親`props`からマシンをカスタマイズするのに便利そうです）。

## 📱 React-Native {#react-native}

[![React DOM for Native (reduce API fragmentation)](/emails/issues/134/rn-web.jpg)](https://github.com/necolas/discussions-and-proposals/blob/reduce-fragmentation/proposals/0000-reduce-fragmentation.md)

**[React DOM for Native (APIフラグメンテーションの削減)](https://github.com/necolas/discussions-and-proposals/blob/reduce-fragmentation/proposals/0000-reduce-fragmentation.md)**

Nicolas Gallagher（Meta / React-Native-Web）による[RFC](https://github.com/react-native-community/discussions-and-proposals/pull/496)が更新されました。この提案の目的は、React-NativeのAPIを可能な限りWeb APIと整合させ、`div`、`span`、さらには`select`のようなDOM要素も含めることです。「一度学べばどこでも動く」が「一度書けばどこでも動く」に変わる可能性があります。コミュニティの意見は分かれています。これが良いアイデアかどうかはもう少し時間をかけて考える必要があり、この野心的なプロジェクトの規模を考えると、実現はまだ先のことでしょう！
