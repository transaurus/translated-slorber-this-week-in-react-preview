## ⚛️ React {#react}

[![Parcel v2.14.0](/emails/issues/226/parcel.jpg)](https://parceljs.org/blog/v2-14-0/)

[**Parcel v2.14.0**](https://parceljs.org/blog/v2-14-0/)

Parcelは非常に興味深い存在です：非常に堅牢で機能が充実したバンドラーです。主要なReactフレームワークでは使用/サポートされていませんが、v2.14.0でいくつかの大きな機能が追加されました：

* クライアント、サーバー、静的レンダリングを備えたServer Components
* Rustライブラリ`mdx-rs`を使用した統合MDXサポート
* `create-react-app`の代替となる`create-parcel` CLI
* 新しいReactエラーオーバーレイ

「独自のフレームワークを構築している」場合やニッチなユースケースがある場合、これらすべてがParcelをさらに魅力的な選択肢にします。

## 📱 React-Native {#react-native}

[![React Native Worklets RFC](/emails/issues/226/worklets.jpg)](https://github.com/software-mansion/react-native-reanimated/discussions/7264)

[**React Native Worklets RFC**](https://github.com/software-mansion/react-native-reanimated/discussions/7264)

React Native Workletsは、Reanimatedから抽出された新しいライブラリで、React Nativeで真のマルチスレッドを実現します。複数のランタイムでJavaScriptコードを実行できるようになります。新しいメモリモデルでは、ランタイム間でデータを扱うために`Serializables`、`Shareables`、`Synchronizables`が導入されています。

マルチスレッドをアニメーションから切り離すことで、Reanimatedはより軽量になり、特にバンドルサイズを小さくしたいウェブアプリにとって有益です。同時に、開発者はアニメーションライブラリに依存せずに、非UIタスク（非同期ストレージ、データベース操作、リアルタイムオーディオ処理、AIを活用した計算など）にマルチスレッドを利用できるようになります。
