## ⚛️ React {#react}

[![Astro 2.0](/emails/issues/132/astro2.jpg)](https://astro.build/blog/astro-2/)

**[Astro 2.0](https://astro.build/blog/astro-2/)**

Astroがv2をリリースしました。主な新機能は以下の通りです：

* コンテンツコレクション：Markdown & MDXの自動型安全性
* ハイブリッドレンダリング：静的？動的？両方の利点を得られます
* 再設計されたエラーオーバーレイ
* 改良された開発サーバー（HMR）
* Vite 4.0
* 公開ロードマップ

また、記事「[Introducing Content Collections: Type-Safe Markdown in Astro 2.0](https://astro.build/blog/introducing-content-collections/)」も参照してください。これはZodを使用してフロントマターを検証し、Reactコンポーネントで型安全にこのデータを使用できるようにし、スキーマを尊重しないフロントマターデータがある場合に良いDXで良いエラーメッセージを表示する方法についてです。ちなみに[Docusaurusも同様のことをしています](https://github.com/facebook/docusaurus/issues/4591) 😜。

[![Remix 1.11](/emails/issues/132/remix0.11.jpg)](https://github.com/remix-run/remix/releases/tag/remix%401.11.0)

**[Remix 1.11](https://github.com/remix-run/remix/releases/tag/remix%401.11.0)**

主な新機能を含む素晴らしいRemixリリースです：

* `defer` + `<Await>` を使用してストリーミングレスポンスを開始。ローダーはクライアントサイドで後から待機されるPromiseを返すことが可能。
* [Route Convention v2](https://remix.run/docs/en/v1/file-conventions/route-files-v2): 新しいルート規約（フラットルート）。現在有効化可能で、v2ではデフォルト規約となる予定。
* CSSインポート、CSS Modules、Vanilla Extractの実験的サポート。

## 📱 React-Native {#react-native}

[![Expo Router RC](/emails/issues/132/expo-router.jpg)](https://blog.expo.dev/announcing-the-expo-router-v1-release-candidate-c5680b88a18c)

**[Expo Router v1 リリース候補版](https://blog.expo.dev/announcing-the-expo-router-v1-release-candidate-c5680b88a18c)**

Expo Routerは、React-Navigationをベースにしたモバイルナビゲーションをファイルシステムの命名規則で作成する提案です。これは多くのReactウェブフレームワークで既に採用されている方法です。レイアウトルート、動的ルート、エラーバウンダリなど、一般的な機能が含まれています。

特に興味深いのは、ユニバーサルサポートです。ナビゲーションはリンクベースで、`<Link>`コンポーネントとMetroのウェブサポート（実験的）を使用しています。Expoのネイティブ部分がうまく機能している今、Expoが徐々にユニバーサルなフルスタックフレームワークになり、[Andrew Clarkの予測](https://twitter.com/acdlite/status/1489755769876754434)を実現する可能性があると感じています。
