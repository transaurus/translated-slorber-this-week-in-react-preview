[![React 18がアプリケーションパフォーマンスを向上させる方法](/emails/issues/152/react18perf.jpg)](https://vercel.com/blog/how-react-18-improves-application-performance)

**[React 18がアプリケーションパフォーマンスを向上させる方法](https://vercel.com/blog/how-react-18-improves-application-performance)**

React 18とそのパフォーマンスへの影響について包括的で分かりやすい記事。Concurrent Reactの仕組みと、Total Blocking TimeやInteraction to Next Paintなどのメトリクスへの影響を解説。Transition API、React Server Components、Suspense、新しいReact cacheなどの新機能も紹介。

[![Kuma UI](/emails/issues/152/kuma.jpg)](https://www.kuma-ui.com/)

**[Kuma UI](https://www.kuma-ui.com/)**

Chakra UIの開発者体験にインスパイアされた新しいCSS-in-JSライブラリで、React Server Componentsと互換性あり。このライブラリは、ゼロランタイムCSS-in-JSと従来のランタイムソリューションの長所を組み合わせた独自のハイブリッド手法を採用。Chakraチームの[Panda CSS](https://panda-css.com/)と比較してどうか、今後の検討が必要です。
