## 📱 React-Native {#react-native}

[![React Native 0.77](/emails/issues/218/react-native-0.77.jpg)](https://reactnative.dev/blog/2025/01/21/version-0.77)

[**React Native 0.77**](https://reactnative.dev/blog/2025/01/21/version-0.77)

予定通り、次のReact Nativeバージョンがリリースされました。主な新機能は以下の通りです：

* CSSサポートの改善：`display: contents`、`boxSizing`、`mixBlendMode`、`outline`
* Android 15のエッジツーエッジと16KBメモリページサイズのサポート（パフォーマンス向上）
* Swift AppDelegateを使用したiOS初期化テンプレート

Expoユーザーにとっての朗報：[SDK 52は既にv0.77をサポート](https://expo.dev/changelog/2025/01-21-react-native-0.77)しており、多くのインフラストラクチャパッケージ（Screens、RNGH、Reanimated）も対応しています。React 19のサポートについてはv0.78を待つ必要がありますが、予想より早くリリースされるかもしれません！

[![Reanimated 4.0 beta](/emails/issues/218/reanimated4.jpg)](https://blog.swmansion.com/reanimated-4-is-new-but-also-very-familiar-b926dd59aa40)

[**Reanimated 4.0 beta**](https://blog.swmansion.com/reanimated-4-is-new-but-also-very-familiar-b926dd59aa40)

Reanimated 4がベータ版となり、これは大きなリリースです！🔥 アニメーションコアが刷新されました。これにより、通常のCSSを使用してネイティブスレッドでアニメーションを簡単に作成できるようになりました。新たに追加されたサポートは以下の通りです：

* CSSプロパティトランジション
* CSSキーフレームアニメーション

これは特にReact Nativeのマイクロインタラクションにおいて、ボイラープレートを大幅に削減し、クロスプラットフォームのアニメーションを実現できる点で非常にエキサイティングです。既存のワークレットAPIも後方互換性を保ち、より高度なニーズに対応できます。ただし、New Architectureが必須となります。

また、[ReactiiveがReanimated v2のアニメーションをv4に移行](https://reactiive.io/articles/css-animations-react-native)した例も参照してください。これは大幅に簡素化されています。そして、古いReanimated v1のコードを移行する話はまだ触れていません🙈。

余談ですが、[現在開発中のReact `<ViewTransition>`コンポーネント](https://motion.dev/blog/reacts-experimental-view-transition-api)をご存知ですか？React-DOMには含まれておらず、おそらく興味深い理由があるでしょう👀。
