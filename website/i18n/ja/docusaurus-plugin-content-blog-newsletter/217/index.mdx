## ⚛️ React {#react}

[![公開：Reactの実験的なアニメーションAPI](/emails/issues/217/viewtransition.jpg)](https://motion.dev/blog/reacts-experimental-view-transition-api)

**[公開：Reactの実験的なアニメーションAPI](https://motion.dev/blog/reacts-experimental-view-transition-api)**

Motionの作成者が、View Transitionsを使用する際のアニメーションのトレードオフと、Reactが新しい`<ViewTransition>`コンポーネントを通じてそれを直接レコンシラーに統合する意義について説明しています。このインタラクティブな記事では、今後のReact機能の早期の様子や現在の実装の詳細の一部を示し、Motionのレイアウトアニメーションと少し重複する点について説明しています。

このエキサイティングな新しいAPIはまだ開発中であり、変更される可能性があり、新しいPRが引き続き追加されています。特に：

* 👀 [React Core PR - View Transitions v2のみをサポート](https://github.com/facebook/react/pull/31996): 新しいコンポーネントは`view-transition-class`などの最近のv2仕様APIを活用し、Safari 18.2+が必要で、Firefoxではまだサポートされていません。
* 👀 [React Core PR - イベント種類に基づくView Transitionクラス名](https://github.com/facebook/react/pull/32050)
* 👀 [Next.js PR - `experimental.viewTransition`フラグ](https://github.com/vercel/next.js/pull/74659): Next.js 15.2 canary 4でこの機能を試すことができます。

## 📱 React-Native {#react-native}

[![EAS Hosting Preview](/emails/issues/217/easdeploy.jpg)](https://expo.dev/blog/expo-announces-eas-hosting-service)

**[EAS Hosting Preview](https://expo.dev/blog/expo-announces-eas-hosting-service)**

Expoは、ウェブの開発者体験（DX）をクロスプラットフォームの世界にもたらすというビジョンを完成させる、初のユニバーサルアプリホスティングプラットフォームを発表しました。これにより、Expoのサーバー関連機能、特にExpo for Web、API Routes、Universal React Server Componentsをシームレスにデプロイできます。モバイルアプリの特殊性（バックエンド/アプリのバージョン同期など）も考慮されています。CLIまたはCI（EAS Workflows）を使用できます。内部ではCloudflare Workersを利用しています。無料プランもあります。
