## ⚛️ React {#react}

[![React Labs - 私たちが取り組んできたこと – 2024年2月](/emails/issues/174/react-labs.jpg)](https://react.dev/blog/2024/02/15/react-labs-what-we-have-been-working-on-february-2024)

**[React Labs - 私たちが取り組んできたこと – 2024年2月](https://react.dev/blog/2024/02/15/react-labs-what-we-have-been-working-on-february-2024)**

今週は、ほぼ1年ぶりにReact Labsからの最初の公式アップデートが届き、いくつかの興奮するような発表がありました。ここですでに多くの内容をカバーしていますが、公式のまとめがあるのは良いことです！

React Forgetは🐦 [今ではReact Compilerと呼ばれ](https://twitter.com/en_JS/status/1758224340880158955)、Instagramでの🐦 [本番環境へのロールアウト](https://twitter.com/sebastienlorber/status/1734607171030303186)が成功したようです。

次のReactリリースはメジャーバージョンになります。React 19が間近に迫っており、その発表にはある[カンファレンス](https://conf.react.dev/)がうってつけでしょう😏。このリリースには、ページメタデータの管理（React-Helmetが提供していた機能の組み込みサポート、適切なストリーミング対応を含む）、アセットローディング（Suspense統合を含む）、クライアントからサーバーへデータを送信するためのActionsなどが含まれます。Next.jsやRemixなどのメタフレームワークが独自実装でカバーする必要のある領域が縮小しています。

最も過小評価されている今後のReact機能の1つであるOffscreen API（Activityという名称）は、React 19に焦点を当てるため一時的に優先度が下げられました。React 18で導入された並行モデルを完全に活用するには、もう少し待つ必要があります。

全体的に見て、React開発者にとってはエキサイティングな時期であり、次のメジャーバージョンを楽しみにしています！

[![React Strict DOM](/emails/issues/174/strict-dom.jpg)](https://github.com/facebook/react-strict-dom)

**[React Strict DOM](https://github.com/facebook/react-strict-dom)**

Metaは今週、Reactに関する大きなニュースを発表しただけでなく、GitHubに新しいリポジトリが登場しました: [react-strict-dom](https://github.com/facebook/react-strict-dom)。READMEには「React DOMとStyleXの実験的な統合で、ウェブとネイティブ向けのスタイル付きReactコンポーネントの開発を改善・標準化することを目的としている」と記載されています。これは[「React DOM for Native」RFC](https://github.com/react-native-community/discussions-and-proposals/pull/496)のアイデアに基づいており、将来的にユニバーサルアプリ開発をどう考えるかについて大きな影響を与える可能性があります。現在、RN向けユニバーサルアプリの最も一般的なアプローチはReact Native Webで、RNコンポーネントとAPIをウェブ上でシミュレートします。RNWは非常に優れていますが、しばしば一方のプラットフォームをもう一方に無理やりはめ込むような感じがします。react-strict-domの前提は、ウェブ、iOS、Android（および将来的にはさらに多くのプラットフォーム）を普遍的にサポートする単一のDOMライブラリが存在するというものです。要素APIはウェブに基づいているため、上記のスクリーンショットで見られる非常にHTMLに似た構文を使用します。

いつものように、Dan Abramovはこのアイデアを簡潔に表現するのが最も優れています 🐦 [ツイート](https://twitter.com/dan_abramov2/status/1760245165703184516):

> 「RNWの精神的な後継者のように見えますが、逆方向からのアプローチです（ウェブAPIが勝ったので、RNは厳格なサブセットのサポートを構築しています）。本質的に、RNで`<div>`をレンダリングできるようになります。」

現在の[互換性ステータス](https://github.com/facebook/react-strict-dom/blob/main/packages/react-strict-dom/COMPATIBILITY.md)を見ると、まだ長い道のりがありますが、これは間違いなくより普遍的なReactへのエキサイティングなマイルストーンです。Metaはすでにこれを使用しています！Lorenzoからのこの 🐦 [スレッドとMicrosoftの関与](https://twitter.com/Kelset/status/1760367655423549480)も参照してください。
