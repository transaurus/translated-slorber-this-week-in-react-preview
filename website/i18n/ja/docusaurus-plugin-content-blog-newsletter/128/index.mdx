**[React BricksはReactコンポーネントベースのビジュアル編集可能なCMSです](https://reactbricks.com/?utm_source=this_week_in_react)**

**開発者にとって柔軟**: Reactコンポーネントを使用して独自のデザインシステムを作成し、JSX内でビジュアル編集を追加し、背景色などのプロパティを編集するためのサイドバーコントロールを追加できます。Next.js、Gatsby、Remix、そしてお好きなCSSフレームワークを選択可能！

**コンテンツ編集者にとって簡単**: デザインシステムを壊すことなく、直接ビジュアルで編集可能。ヘッドレスCMSのような抽象的なフィールドでの編集よりもはるかに簡単です。**そして時間を節約**できます。コンテンツ編集者が自律的に作業できるからです！

**エンタープライズ対応**: コラボレーション、タイムマシン、シングルサインオン、GDPR準拠のデータセンター、最適化された画像のためのグローバルCDN、Eコマース統合、スケジュールされた公開など。

## ⚛️ React {#react}

[![SWR 2.0](/emails/issues/128/swr2.png)](https://swr.vercel.app/blog/swr-v2)

**[SWR 2.0](https://swr.vercel.app/blog/swr-v2)**

Vercelのデータフェッチングライブラリの新しいメジャーリリースです。以下のようなさまざまな改善が行われました:

* 新しい`useSWRMutation`フック
* 簡素化されたOptimistic UI
* ブラウザ拡張: [SWR DevTools](https://swr-devtools.vercel.app/)
* 命令型のプリロードAPI
* React 18サポートの改善

[![Vite 4](/emails/issues/128/vite4.png)](https://vitejs.dev/blog/announcing-vite4.html)

**[Vite 4](https://vitejs.dev/blog/announcing-vite4.html)**

Viteは人気を集め続けているフロントエンドツールです。この最新バージョンではRollup 3が採用され、Viteの内部アセット管理が大幅に簡素化されました。現在、2つのReactプラグインが異なるトレードオフで提供されています:

* [vite-plugin-react](https://github.com/vitejs/vite-plugin-react): esbuild + Babel
* [vite-plugin-react-swc](https://github.com/vitejs/vite-plugin-react-swc): 開発時にBabelをSWCに置き換え、コールドスタートとHMRが改善されたようです

[![Codux - Reactプロジェクト向けビジュアルIDE](/emails/issues/128/codux.jpg)](https://dev.to/codux/introducing-codux-15j5)

**[Codux - Reactプロジェクト向けビジュアルIDE](https://dev.to/codux/introducing-codux-15j5)**

NadavはWixの共同創業者であり、React（そして近い将来他のフレームワークも）向けの新しいプロジェクトであるビジュアルIDEを紹介しています。これは現在のIDEを置き換えるものではなく、補完することを目的としています。Coduxでは、コンポーネントを隔離して作業でき、ビジュアルインターフェースを介してpropsやスタイルを編集することが可能です。[Meet Codux](https://www.youtube.com/watch?v=KG6i6cCz0P0)という動画を見ると理解が深まるでしょう。Storybookに似ていますが、編集機能があり、コードとの双方向同期が可能です。
