## ⚛️ React {#react}

[![Sathya Gunasekaran - React Compilerの今後](/emails/issues/208/compiler.jpg)](https://www.youtube.com/watch?v=qd5yk2gxbtg)

🎥 [**Sathya Gunasekaran - React Compilerの今後**](https://www.youtube.com/watch?v=qd5yk2gxbtg)

この素晴らしい講演で[Reactコンパイラのベータ版](https://react.dev/blog/2024/10/21/react-compiler-beta-release)が発表されただけでなく、他にも興味深い情報が公開されました。コンパイラは自動メモ化だけでなく、さらなる最適化を構築するためのプラットフォームです。紹介された3つの今後の機能：

* 型設定: 追加の型情報を提供することで、コンパイラの最適化を支援できます。
* JSXインライン化: コンパイラは不要なJSXランタイム呼び出しを削除し、事前ビルドされたJSXオブジェクトに置き換えることが可能です。`jsx('div')`が`{type: 'div'}`になります。
* JSXアウトライン化: コンパイラはサブコンポーネントを自動的に抽出し、独立して最適化できます。これは特に`array.map`とインラインJSXを使用するリスト要素で有用で、コンパイラは`<Item>`コンポーネントを作成できます（トークのスクリーンショットを参照、または[このPR](https://github.com/facebook/react/pull/30956)で詳しく説明されています）。

## 📱 React-Native {#react-native}

[![React Navigation 7.0](/emails/issues/208/reactnavigation7.jpg)](https://reactnavigation.org/blog/2024/11/06/react-navigation-7.0/)

[**React Navigation 7.0**](https://reactnavigation.org/blog/2024/11/06/react-navigation-7.0/)

React Navigation v7がリリースされ、Expo SDK 52で提供予定のExpo Router v4（現在ベータ版）をサポートします。主なハイライトは以下の通りです：

* スクリーンプリロード：ナビゲートする前にスクリーンを事前に準備でき、体感パフォーマンスが向上します。これは非常にエキサイティングで、将来的にはConcurrent Reactの機能を活用できる可能性があります。
* 静的API：ナビゲーター設定が簡素化され、TypeScriptやディープリンクとの連携が容易になります。
