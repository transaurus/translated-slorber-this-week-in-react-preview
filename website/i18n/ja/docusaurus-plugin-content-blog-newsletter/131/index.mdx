## ⚛️ React {#react}

[![useReducer](/emails/issues/131/useReducer.jpg)](https://www.builder.io/blog/use-reducer)

**[useState地獄の治療法](https://www.builder.io/blog/use-reducer)**

複数の`useState`を単一のアクションを持つ`useReducer`に置き換えることを提案しています。状態が一貫していることを確認できる利点や、後で複数のアクションを持つリデューサーに移行できる利点を説明しています。知っておくと便利なパターンです。私の意見：単一のオブジェクトとカスタムフックを使用した単一の`useState`でも同様の利点を得ることが可能です。`useState`か`useReducer`かは実装の詳細であるべきです。

[![useReducer](/emails/issues/131/react-query-ts.jpg)](https://tkdodo.eu/blog/type-safe-react-query)

**[タイプセーフなReact Query](https://tkdodo.eu/blog/type-safe-react-query)**

TypeScriptでは、自分自身を騙したくないものです。バックエンドが正しいペイロードを返すと信じられるでしょうか。DominikはReact-QueryとZodを組み合わせることを提案しています。推論を使用すると、`as`やジェネリクスで型を強制する必要がなくなります。バックエンドとフロントエンドが同じモノレポにあるフルスタック環境では、tRPCやZodiosを使用できます。
