## ⚛️ React {#react}

[![React Server Components向けDevtools](/emails/issues/155/rscdevtools.jpg)](https://www.alvar.dev/blog/creating-devtools-for-react-server-components)

[**React Server Components向けDevtools**](https://www.alvar.dev/blog/creating-devtools-for-react-server-components)

React Server Componentsのリリース後、Alvarはクライアントとサーバー間のネットワーク通信に解読が難しいフォーマットが使用されていることに気づきました。そこで彼は、ペイロードをより便利に検査できるウェブサイト[RSC Parser](https://rsc-parser.vercel.app/)の開発を開始しました。これは機能しましたが、UXは理想的ではありませんでした。その後、彼はChromeのNetwork DevToolsのように機能する本格的なブラウザ拡張機能の開発に取り組みました。このツールには、時間経過とともにペイロードを表示するタイムラインや、Suspenseを使ったストリーミングサポートも含まれています。このツールにより、彼は自身のブログで過度に大きなペイロードを検出し、最適化することができました。

これはまだ開発中の有望なツールで、注目に値します。拡張機能はまだブラウザストアでは利用できませんが、近日中に公開される予定です。この[Twitterスレッド](https://twitter.com/sebastienlorber/status/1689203357368365056)でいくつかのデモ動画を確認できます。

[![Next.jsのキャッシュ](/emails/issues/155/cachinginnextjs.jpg)](https://nextjs.org/docs/app/building-your-application/caching)

[**Next.jsのキャッシュ**](https://nextjs.org/docs/app/building-your-application/caching)

Next.jsの新しいドキュメントページでは、4つのキャッシュレベル（Request Memoization、Data Cache、Full Route Cache、Router Cache）がどのように機能し、なぜ存在するのか、そしてそれらをバイパスする方法について包括的に説明されています。

* リクエストメモ化
* データキャッシュ
* フルルートキャッシュ
* ルーターキャッシュ

Routerキャッシュをバイパスすることはできません。この問題は[この動画](https://www.youtube.com/watch?v=_yhSh4g0NSk)で指摘されています。

## 💸 Sponsor {#sponsor2}

[![Google Formsに代わる非クラウド型の包括的ソリューション](/emails/issues/155/surveyjs.jpg)](https://surveyjs.io/?utm_source=thisweekinreact\&utm_medium=email)

**[Google Formsに代わる非クラウド型の包括的ソリューション](https://surveyjs.io/?utm_source=thisweekinreact\&utm_medium=email)**

データフローを完全に制御可能な堅牢なフォーム管理システムを構築したい場合、そしてその導入に何ヶ月も数千ドルも費やしたくない場合、SurveyJSが最適な選択肢です！

SurveyJSは4つのオープンソースJavaScriptライブラリから成るプロダクトスイートで、ドラッグ＆ドロップのフォームビルダーで動的なJSONベースのフォームを作成・編集し、Reactアプリでレンダリングし、機密性の高い調査データを自社インフラ内に保存できます。さらに、カスタム調査をブラウザ上で編集可能なPDFファイルとして表示したり、インタラクティブなチャートやテーブルで調査結果を可視化したりできます。[詳細はこちら](https://surveyjs.io/?utm_source=thisweekinreact\&utm_medium=email)。
