## ⚛️ React {#react}

[![Next.js PR - Node.jsミドルウェアのサポート](/emails/issues/220/nextjs-middleware.jpg)](https://github.com/vercel/next.js/pull/75624)

**[Next.js PR - Node.jsミドルウェアのサポート](https://github.com/vercel/next.js/pull/75624)**

Next.jsコミュニティ待望の機能です。最新のv15.2 canaryには、`experimental.nodeMiddleware`フラグが追加され、`runtime: ‘nodejs’`を選択することで、ミドルウェア内でNode.js APIを使用できるようになり、エッジランタイムと互換性のあるAPIに制限されなくなります。

Vercelミドルウェアも[ゼロから再構築され](https://x.com/galstar/status/1886832573482934706)、複数のランタイムをサポートし、[Fluid compute](https://vercel.com/blog/introducing-fluid-compute)上で動作するようになりました。これは、Vercelがサーバーレス関数を実行するための新しいコスト効率の良い方法です。
