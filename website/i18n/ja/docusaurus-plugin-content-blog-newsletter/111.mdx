## React

[**Reassure**](https://github.com/callstack/reassure?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Reassureは新しいReactパフォーマンステストライブラリです。以前紹介されましたが、Callstackによってオープンソース化されました。パフォーマンスの退行が検出された場合、プルリクエストにレポートを投稿できます。例：以前より再レンダリングが遅い、または頻繁なコンポーネント。現時点ではReact-Nativeのみサポートされていますが、Webサポートも間もなく追加予定です。

[**マルチプレイヤーカーソルのアニメーション方法**](https://liveblocks.io/blog/how-to-animate-multiplayer-cursors?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

複数のブラウザでマウスカーソルをリアルタイムにアニメーションさせるインタラクティブな記事です。見た目以上に複雑な問題です。各アプローチはReact実装で説明され、CSSトランジション、スプリング、スプラインなど異なるトレードオフを示しています。

[**Next.jsミドルウェアの使い方**](https://www.ctnicholas.dev/articles/how-to-use-nextjs-middleware?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

同じ著者によるミドルウェアのユースケースを紹介するインタラクティブ記事です。以前ここで紹介されましたが、Next.js 12.2の正式リリースに伴う破壊的変更を反映して更新されました。

[**10 Years Of Meteor**](https://meteor10.sachagreif.com/?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Sacha Grief（『Discover Meteor』の共著者であり、State of JSやCSSの調査でも知られる）が、自身とMeteorフレームワークとの10年にわたる関わりを振り返ります。Meteorの人気は下降しましたが、今も利用されており、フロントエンドエコシステムに大きな影響を与えました。

[**The mystery of React Element, children, parents and re-renders**](https://www.developerway.com/posts/react-elements-children-parents?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Nadiaが、Reactの驚くべき、あるいは直感に反するような挙動を指摘し、さまざまなエッジケースにおいて、親コンポーネントの再レンダリングが子コンポーネントの再レンダリングを引き起こす条件について解説します。

[**Storybook Community Showcase #2**](https://storybook.js.org/blog/community-showcase-2/?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Storybookコミュニティの最新ニュースのまとめ：Component Encyclopedia、Figmaプラグイン、Story Explorer、そして各種アドオン（Variants、CSS variables、Recoilなど）が紹介されています。

[**The new wave of React state management**](https://frontendmastery.com/posts/the-new-wave-of-react-state-management/?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

状態管理ライブラリが解決すべきさまざまな問題を提示し、いくつかのライブラリを比較します。Redux以前と以後の回顧も含まれています。内容は充実していますが、かなり長く、読みやすいとは言えません。

## React-Native

[**FlashList - 高速で高性能なReact Nativeリスト**](https://shopify.github.io/flash-list/?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Shopifyは、スクロール時のパフォーマンス問題を解決し、空のセルを表示しないようにするためのFlatListの代替としてFlashListをリリースしました。目標は、低スペックのAndroidデバイスでも60fpsを維持しつつ、実装を複雑にしないことです。このライブラリは既にレンダリングされたセルを再利用します。Reactの`key`の使い方を調整する必要があります。

[**React-Native-Compat-Table**](https://retyui.github.io/react-native-compat-table/?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

各React-Nativeランタイム（JSC、V8、Androidなど）のAPIサポートを包括的にリストアップしています。著者はまた、[なぜReact NativeがTS設定lib: "es2017"を推奨するのか](https://dev.to/retyui/why-react-native-typescript-config-recommends-to-use-a-five-years-old-standard-lib-es2017-3bjk?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)について説明しています。
