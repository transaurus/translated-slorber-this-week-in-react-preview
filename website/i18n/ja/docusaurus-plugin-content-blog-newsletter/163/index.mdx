## ⚛️ React {#react}

[![TanStack Query v5](/emails/issues/163/tanstack-query-v5.jpg)](https://tanstack.com/blog/announcing-tanstack-query-v5)

[**TanStack Query v5**](https://tanstack.com/blog/announcing-tanstack-query-v5)

多くのプレリリースを経て、TanStack Query v5.0の安定版がリリースされました。React Queryパッケージも含まれています。特に以下の機能が含まれています：

* 統一された、より型安全なAPI（オーバーロードなし）
* 安定版のSuspenseサポート。ストリーミング、React Server Components（実験的）
* その他多くの改善：無限クエリ、永続化、開発者ツール、楽観的更新、useMutationStateなど

[![Rspress v1.0 - Rspackベースの高速静的サイトジェネレーター](/emails/issues/163/rspress.jpg)](https://github.com/orgs/web-infra-dev/discussions/5)

[**Rspress v1.0 - Rspackベースの高速静的サイトジェネレーター**](https://github.com/orgs/web-infra-dev/discussions/5)

ByteDanceチームによる新しい静的サイトジェネレーターの最初の安定版リリースです。Rust、React、MDXを基盤としており、Docusaurus、Nextra、Vitepress、Starlightなどのコンテンツ指向ソリューションにとっての強力な競合です。Rspressは高性能でありながら、i18n、バージョニング、検索など、印象的な機能リストを提供します。競争が激化していますが、それがDocusaurusにRustを導入するモチベーションをさらに高めてくれています😄！

[![Remix - Viteサポート](/emails/issues/163/remix-vite.jpg)](https://remix.run/docs/en/dev/future/vite)

[**Remix - Viteサポート（不安定版）**](https://remix.run/docs/en/dev/future/vite)

Remixはこの[提案](https://github.com/remix-run/remix/discussions/7632)で、Viteプラグイン化する決定について説明しています。これによりコンパイラを開放し、ユーザーが独自のViteプラグインを追加できるようになることで柔軟性が向上し、MDXやGraphQL、Tailwind、パスエイリアスなど様々なシナリオに対応可能になります。

[![Reactコンポーネントの静的解析を楽しみながら利益を得る](/emails/issues/163/react-forget.jpg)](https://www.youtube.com/watch?v=kjOacmVsLSE)

🎥 [**Reactコンポーネントの静的解析を楽しみながら利益を得る**](https://www.youtube.com/watch?v=kjOacmVsLSE)

Sathya氏は待望のコンパイラ「React-Forget」の開発に取り組んでいます。この講演ではいくつかの興味深い詳細が明かされています：

* コンパイルが可能なのは、Reactの新しいモデルがクラスや`this`を使用しなくなったため
* ローカルに推論し、並列で最適化でき、段階的に導入可能
* 型なしJavaScriptでも動作し、フックや"use"プレフィックスといったReactの慣習を理解
* エイリアス解析システムはV8エンジンよりもさらに高度

Sathya氏によると、今後数週間でReact-Forgetに関するさらなる情報が公開される予定で、2日後に開催されるReact Advancedカンファレンスの["Understanding Idiomatic React"講演](https://reactadvanced.com/schedule-offline)でも触れられるそうです。要するに、待ちきれないほど楽しみな内容です！
