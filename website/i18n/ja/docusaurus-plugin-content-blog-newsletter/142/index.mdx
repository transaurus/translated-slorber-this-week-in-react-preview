## ⚛️ React {#react}

[![TypeScript 5.1 beta](/emails/issues/142/ts5.1.jpg)](https://devblogs.microsoft.com/typescript/announcing-typescript-5-1-beta/)

**[TypeScript 5.1 (ベータ) + React](https://devblogs.microsoft.com/typescript/announcing-typescript-5-1-beta/)**

Reactが独自の有効なJSX型を宣言できるようにする重要なTypeScriptリリースです。Reactの型定義では、この新機能を可能な限り早く使用する予定で、すでに[フォーク](https://github.com/DefinitelyTyped/DefinitelyTyped/pull/65126)されて古いバージョン（TypeScript ≦ 5.0）のサポートを継続しています。

TypeScript 5.1+をできるだけ早く採用するには良い理由があります。特に以下のようなことが可能になります：

* React Server Componentを`async`でJSX内で使用してもTypeScriptエラーや[回避策](https://gist.github.com/cramforce/b5e3f0b103f841d2e5e429b1d5ac4ded)が不要になります。
* `ReactNode`として有効なすべての型（文字列、配列、booleanなど）をフラグメントなしで返せるようになります（[詳細](https://twitter.com/sebastienlorber/status/1648619398259769344)）。
* JSX周りのその他の興味深い改善点：名前空間付き属性とリンクされた編集（VSCode）。

[![意図的にReact QueryのAPIを破壊する](/emails/issues/142/rq.jpg)](https://tkdodo.eu/blog/breaking-react-querys-api-on-purpose)

**[意図的にReact QueryのAPIを破壊する](https://tkdodo.eu/blog/breaking-react-querys-api-on-purpose)**

React-Queryはv5で`useQuery`フックからコールバック（`onSuccess`、`onError`、`onSettled`）を削除します。Dominikはこの選択の理由を説明し、トースト表示などの一般的なケースに適した代替案を提案しています。これらのAPIが誤用されることが多かったため、議論を呼ぶものの必要な決定でした。注：`useMutation`ではコールバックは保持されます。
