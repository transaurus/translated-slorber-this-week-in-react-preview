## ⚛️ React {#react}

**[Vercel OG Image Generation: エッジでの高速で動的なソーシャルカード画像生成](https://vercel.com/blog/introducing-vercel-og-image-generation-fast-dynamic-social-card-images)**

Vercelは、Vercel Edge Functionsで動的なソーシャルカードを生成するための新しい非常に効率的なソリューションを発表しました。実用的な点として、追加の設定なしでJSX、Reactコンポーネント（ステートレス）、およびTailwindを使用できます。これらがどのように機能するかを説明する[このスレッド](https://twitter.com/shuding_/status/1579607964549513217)をお勧めします。非常に興味深いです！彼らはReact-NativeのレイアウトエンジンであるYogaさえ使用しています 🤭

**[Reactにおけるレンダリングの未来](https://prateeksurana.me/blog/future-of-rendering-in-react/)**

React 18以降のレンダリングパターンをよく説明した記事です。SSRストリーミングの仕組み、Suspense、React lazy、Server Componentsでの機能フラグの実装などについての情報が得られます。

**[Reactプロジェクトでよくある問題の克服](https://dinosaurs-with-jetpacks.com/posts/react-problems)**

JavaScriptアプリをフレームワーク（Reactなど）から切り離すことを推奨しています。よりシンプルなアプローチへの回帰を促しています。賛同を得られないかもしれないいくつかの意見も含まれています。

**[StorybookでのMaterial UI](https://storybook.js.org/blog/material-ui-in-storybook/)**

特に興味深いセクションを含むチュートリアルで、基盤となるUIライブラリのTypeScriptタイプを再利用してStorybookコントロールを自動生成する方法を説明しています。プロパティを転送する際に非常に便利です。`Pick`や`Omit`のようなヘルパーもサポートしています。

* 👤 Sathya Gunasekaran（元V8）が[Reactコアチームに加入](https://twitter.com/_gsathya/status/1578816177371250688)、おそらく[React-Forgetコンパイラの作業に携わる](https://twitter.com/Huxpro/status/1578865666010263552)予定
* 👥 [Next.js Conf 2022のスピーカー](https://twitter.com/vercel/status/1578401243243741185)
* 📖 ベータ版ドキュメント: [React.Children](https://twitter.com/dan_abramov/status/1578457525628047360) + [useId](https://twitter.com/dan_abramov/status/1577283961763467265) + [createElement](https://twitter.com/dan_abramov/status/1577463721193897985) + [useImperativeHandle](https://twitter.com/dan_abramov/status/1577717226269966336)
* 📖 [Next.js on Vercelドキュメントの改善](https://twitter.com/leeerob/status/1577788730349436928)
* 🌶 [ReactとNext.jsは終わった――ついに（完全に）置き換わる新たな何か](https://javascript.plainenglish.io/react-and-next-js-is-dead-something-new-is-finally-replacing-it-for-good-c792c48806f6): QwikフレームワークがReactよりも優れたモデルを提供する方法を解説。釣りタイトルだが、非常に興味深い記事。
* 📜 [Reactカスタムレンダラーの構築 - パート1](https://www.markcodes.dev/posts/build-react-custom-renderer-part-1): Reactレンダラーを作成する方法を示す新シリーズ。現時点では追いやすい内容だが、更新機能はまだサポートされていない。
* 📜 [Remix、React、そして状態管理](https://blog.jim-nielsen.com/2022/remix-react-and-state/): Remixのメンタルモデルを、エンドツーエンド/フルスタックな方法でReactの状態と調和させる。
* 📜 [独自の状態管理ライブラリの書き方](https://judehunter.dev/blog/how-to-write-your-own-state-management-library): 状態管理ライブラリの動作原理と、セレクターシステムの実装方法を理解するのに有用。
* 📜 [パフォーマンスを考慮したReactでのデータ取得方法](https://www.developerway.com/posts/how-to-fetch-data-in-react): Reactでのデータ取得が単純ではない理由を解説。パフォーマンスはUX/知覚に関連。ウォーターフォール、ブラウザのリクエスト制限など...
* 📜 [React、TypeScript、xStateで不可能なUI状態を回避](https://whereisthemouse.com/avoid-impossible-ui-states-with-react-typescript-and-xstate): TypeScriptの判別共用体型は不可能な状態を回避し、XStateは不可能な状態遷移を防ぐ。
* 📜 [新しいJavaScriptフレームワーク作成について考えを改めた](https://whitep4nth3r.com/blog/write-a-new-javascript-framework/): SSG + SSRの両方をサポートするフレームワークは4つだけ - Eleventy、Next.js、Gatsby、Redwood。
* 📜 [Sandpackをコードデモに活用する方法](https://www.ctnicholas.dev/articles/how-to-use-sandpack-for-code-demos): [最新のSandpack機能で更新された記事](https://twitter.com/ctnicholasdev/status/1577683748878884864)、ライブコードプレイグラウンドを作成するこのCodesandboxツール。Reactベータ版ドキュメントやJosh Comeauなどで使用。
* 📦 [Tremor 1.0](https://www.tremor.so/): ダッシュボード分析を作成するためのReact UIコンポーネントライブラリ。
* 🐦 [W3C Trusted Types + React + XSS](https://twitter.com/sebastienlorber/status/1579485631260286977): 新しいHTML提案はXSS脆弱性を防ぐネイティブソリューションを提供し、将来的には`dangerouslySetInnerHTML`の必要性をなくすかもしれない？
