## ⚛️ React {#react}

[![Base UI 1.0 alpha](/emails/issues/214/base-ui.jpg)](https://base-ui.com/)

[**Base UI 1.0 alpha**](https://base-ui.com/)

私は通常UIライブラリには興味がありません。しかし、Radix、MUI、Floating UIの開発者たちが集まって、25のスタイルなしでアクセシブルで構成可能なプリミティブコンポーネントを作成し、細部までこだわっているのを見ると、注目する価値があると思います！Radixは現在非常に人気があります（特にshadcn/uiのおかげで）が、最近は[メンテナンスがやや減少](https://github.com/radix-ui/primitives/graphs/contributors)しており、新しいヘッドレスライブラリの登場は常に歓迎です！

**React 19とカスタム要素**

興味深い関連記事をいくつか見つけました：

* 📜 [React 19とWeb Componentの例](https://frontendmasters.com/blog/react-19-and-web-component-examples/): Chris CoyierがReact 19のカスタム要素サポートをテストし、Lit要素を作成してJSX経由でレンダリングしています。オブジェクトをプロパティとしてシームレスに渡せますが、イベントハンドラにはカスタムイベントのディスパッチが必要な場合があります。
* 📜 [React 19がカスタム要素の完全サポートを導入](https://aleks-elkin.github.io/posts/2024-12-06-react-19/): React 18とReact 19のサポート改善を比較しています。
* 📜 [Sweet Suspense](https://plainvanillaweb.com/blog/articles/2024-09-09-sweet-suspense/): Reactのlazy/Suspense/ErrorBoundaryをカスタム要素で再現する楽しい実験です。
* 📜 [Isomorphic Web Components](https://jakelazaroff.com/words/isomorphic-web-components/): Reactは使用していませんが、Web Componentsのサーバーサイドレンダリングサポートについての有用な概要です。
