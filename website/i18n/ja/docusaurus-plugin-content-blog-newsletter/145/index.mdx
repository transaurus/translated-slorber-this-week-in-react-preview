## ⚛️ React {#react}

[![Next.js 13.4](/emails/issues/145/nextjs.jpg)](https://nextjs.org/blog/next-13-4)

**[Next.js 13.4](https://nextjs.org/blog/next-13-4)**

Next.jsのメジャーリリースで、以下の機能が含まれています：

* App Router（安定版）：SuspenseとServer Componentsに基づく新しいルーターが本番環境に対応
* Turbopack（ベータ版）：新しいバンドラーが開発環境で安定化し、本番ビルドが間もなくリリース予定
* Server Actions（アルファ版）：最新のReactの革新をNext.jsに統合

ブログ記事では、旧ルーターとの多くの注目すべき違いが示されています：規約、ネストされたレイアウト、選択的なハイドレーション、サーバーコンポーネント、コード分割など...

また、サーバーとのインタラクションを大幅に簡素化する、Reactの最新の革新であるServer ActionsがNext.jsに統合されました。これにより、APIルートを作成する必要がなく、[プログレッシブエンハンスメント](https://twitter.com/acdlite/status/1654159613950828546)をサポートするフォームを使用せずに、フルスタックのReactコンポーネントを作成および配布できるようになりました。

ただし、Server Actionsを本番環境で採用するにはまだ少し早いです。アルファ版であり、Server Actionのクロージャに含まれる秘密がクライアントに意図せず公開される可能性があります。詳細はこちら：

* 🧵 [スレッド](https://twitter.com/tomus_sherman/status/1654198731313500174): この問題について警告
* 🐦 [Sebastian Markbåge](https://twitter.com/sebmarkbage/status/1654212899097960448) と [Andrew Clark](https://twitter.com/acdlite/status/1654317374777032704) によって認識され、変数の暗号化が提案されています。
* 🎥 [NextのServer Actionsはそれほど安全ではないかもしれない...](https://www.youtube.com/watch?v=fdKcbyEK66M)
