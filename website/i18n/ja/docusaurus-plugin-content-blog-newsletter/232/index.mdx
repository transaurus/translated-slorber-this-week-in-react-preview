## ⚛️ React {#react}

* 💸 [ConvexがMCPサーバーをリリース](https://convex.link/thisweekmcp)
* 🔐 [React Routerの脆弱性とv7.5.2での修正](https://github.com/remix-run/react-router/blob/main/CHANGELOG.md#v752): React Router v7にはDOS攻撃やXSS攻撃につながる重要なセキュリティ脆弱性が2つ存在していました。これらは最新のv7.5.2リリースで修正されました。Vercel、Netlify、Cloudflareなどのクラウドプロバイダーは既に対策を展開していますが、アップグレードしておくことが推奨されます。
* 👀 [Next.jsドキュメントPR - Vercel言及の削除](https://github.com/vercel/next.js/pull/78557): Next.jsのドキュメントからVercelへの偏りが減り、競合他社がより公平に扱われるようになります。個人的には、OSSソフトウェアをコミュニティに提供し、有料サービスのマーケティング手段として活用すること（多くのOSSプロジェクトが持続可能になる方法）に問題はないと考えます。Vercelのような大企業でもこれはフェアな戦略ですが、失われた信頼を回復する必要があるでしょう。なお、VercelがReactチームを「乗っ取る」わけではなく、先週発表されたクライアントサイドのみのReact Labs機能（`<ViewTransition>`など）は[Vercel社員によって実装されています](https://bsky.app/profile/ricky.fm/post/3lnvyuennss2b)。
* 👀 [Vite RSC計画](https://github.com/hi-ogawa/vite-plugins/issues/748): Viteコントリビューターが新規パッケージ`react-server-dom-vite`の開発を[引き継ぎ](https://github.com/facebook/react/pull/31768#issuecomment-2823466684)、WakuやRedwoodSDKなどのフレームワーク向けに統合PRが多数送られています。
* 🗓 [React-Summit](https://reactsummit.com/?utm_source=Newsletter\&utm_medium=thisweekinreact) - 🇳🇱 アムステルダム - 6月13日＆17日。世界最大のReactカンファレンスで60人以上のスピーカーが登壇！チケットはプロモコード「TWIR」で10%オフ。
* 🗓️ [StrapiConf](https://conf.strapi.io/?utm_source=thisweekinreact) - 🇫🇷 パリ - 5月13日。ヘッドレスWeb開発の未来を深掘りする最後のチャンス！コード「TWIR」で10%割引。
* 📜 [「use client」は何をするのか？](https://overreacted.io/what-does-use-client-do/): Dan AbramovがRSCにはAPIサーフェスがなく、ディレクティブは2つの世界をつなぐモジュールシステムの機能であり、Reactを超えて意義を持つ可能性があると説明。Danは[YouTubeチャンネルを再開](https://www.youtube.com/@overreacted-io)したので、ブログ記事の解説動画もチェックできます。
* 📜 [ReactでPromiseをシリアライズできる](https://twofoldframework.com/blog/you-can-serialize-a-promise-in-react): サーバーからクライアントへPromiseをストリーム経由で転送するReactの仕組みを解説したインタラクティブ記事。このメカニズムのミニチュア版を再構築する方法を示しています。
* 📜 [React内部: どのuseEffectが最初に実行される？](https://frontendmasters.com/blog/react-internals-which-useeffect-runs-first/): `useEffect`コールバックは最深の子からルートへボトムアップで実行される（`<Activity>`使用時は別）という良い復習。

## 📱 React-Native {#react-native}

[![React Native Enterprise Framework](/emails/issues/232/rnef.jpg)](https://www.rnef.dev/)

**[React Native Enterprise Framework](https://www.rnef.dev/)**

Callstackプロジェクトがオープンソース化されました。これは既存のモバイルアプリ（「ブラウンフィールド」と呼ばれる）にReact Nativeを段階的に導入するためのRNコミュニティCLIの代替として設計されています。主な機能は以下の通りです：

* 簡素化されたブラウンフィールドアプローチによる段階的な導入
