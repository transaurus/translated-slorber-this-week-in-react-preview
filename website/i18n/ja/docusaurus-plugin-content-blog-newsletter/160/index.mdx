## ⚛️ React {#react}

[![Remix v2](/emails/issues/160/remix-v2.jpg)](https://remix.run/blog/remix-v2)

[**Remix v2**](https://remix.run/blog/remix-v2)

予想通り、Remix v2は比較的スムーズなアップグレードとなるでしょう。多くのv2機能はv1でオプトインの「future flags」としてすでに利用可能でした。この記事では、v1で提供され、v2でデフォルトで有効になったすべての機能の概要を説明しています。Remixはまた、v1の慣習を好むユーザーのために後方互換性パッケージを提供しています。React Server ComponentのサポートはRemix v3で実装される予定です。

## 📱 React-Native {#react-native}

[![Expo Router RFC - APIルート](/emails/issues/160/expo-api-routes.jpg)](https://blog.expo.dev/rfc-api-routes-cce5a3b9f25d)

[**Expo Router RFC - APIルート**](https://blog.expo.dev/rfc-api-routes-cce5a3b9f25d)

待ち望んでいた機能です！Expoは、Expo Routerの規約を通じてAPIルートをデプロイするサービスを提供することで、サーバーサイド機能に参入します。これらのルートは自分でデプロイできますし、近い将来人気のあるホストでもデプロイ可能になりますが、長期的な目標はサーバーランタイムをEASでホストすることを推奨することです。

これはExpoにとって新たな時代の始まりです。ネイティブモバイルの問題を解決した今、サーバーサイドレンダリングとReact Server Componentsでウェブサポートのギャップを埋めることで、クロスプラットフォームを主流の現実にする時が来ました。
