[![Vercelのストレージ紹介](/emails/issues/144/vercel.jpg)](https://vercel.com/blog/vercel-storage)

**[Vercelのストレージ紹介](https://vercel.com/blog/vercel-storage)**

VercelはReact Server ComponentsやNext.jsでの使用に特に適した、エッジファーストのストレージサービスをリリースしました：

* Vercel KV：Upstashベースのキーバリューストア（Redis）

* Vercel Blob：Cloudflare R2ベースのファイルストレージ

現在[Vercel Ship week](https://vercel.com/ship)開催中で、今後数日でさらにニュースが発表される予定です。選ばれた画像から、明日には**Next.js Server Actions**という新機能を導入したNext.jsのリリースがあると推測されます。これはまた、Remixと非常に似た方法でReactアプリの[プログレッシブエンハンスメント](https://twitter.com/sebastienlorber/status/1653701538336915456)を改善する意図も明らかにしています：このフォームはJavaScriptやハイドレーションなしでも動作します。

注: この例はテンプレートリテラルを使用しているため、SQLインジェクションの脆弱性はありません。

[![Million.js - Virtual DOM: Back in Block](/emails/issues/144/million.jpg)](https://millionjs.org/blog/virtual-dom)

**[Million.js - Virtual DOM: Back in Block](https://millionjs.org/blog/virtual-dom)**

Million.jsとその「ブロック仮想DOM」コンセプトを紹介するインタラクティブ記事。差分検出のアプローチが異なり、非再帰的で静的解析、コンパイル、ダーティチェックに基づいており、Svelteと非常に似ています。

Million.jsはReactを置き換えるものではなく、パフォーマンス向上のためにReactと統合されます。非常に特定のケースでのみ使用が推奨されます：多くの静的コンテンツと安定した構造のUIツリーがある場合です。

## 📱 React-Native {#react-native}

[![React Native macOS 0.71の紹介](/emails/issues/144/rnmacos.jpg)](https://devblogs.microsoft.com/react-native/2023-04-27-announcing-macos-71/)

**[React Native macOS 0.71の紹介](https://devblogs.microsoft.com/react-native/2023-04-27-announcing-macos-71/)**

Microsoftはv0.69とv0.70のリリースをスキップし、直接v0.71に移行しました。現在の目標は、他のプラットフォームと同期を保つことです：v0.72の作業が進行中です。このプラットフォームバージョンの統一は非常に有用です：フィードバックによると、単一のモノレポ内で異なるReact-Nativeバージョンを混在させるのは容易ではありません。

新しくリリースされたv0.71では、Fabricの実験的サポートや、他のプラットフォームで最近リリースされたすべての新機能（TypeScript、Flexboxなど）が導入されています。MicrosoftとMetaは協力しており、このプラットフォームの使用は両社内で拡大しています：例えば、MetaはMessenger DesktopでReact-Native macOSを使用しています。
