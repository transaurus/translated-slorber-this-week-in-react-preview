## ⚛️ React {#react}

[![TanStack Router](/emails/issues/123/tanstack-router.png)](https://tanstack.com/router/v1)

**[TanStack Router](https://tanstack.com/router/v1)**

Tannerが新しいプロジェクトを発表しました：100%タイプセーフで、フレームワークに依存しないルーターです。以下のような印象的な機能が含まれています：

* URL状態管理
* 組み込みキャッシュ
* 検索パラメータのスキーマ/検証
* 自動プリフェッチ
* ローダー/アクション
* 非同期要素 + コード分割

一見すると、RemixやReact-Routerからのインスピレーションがあるように見えますが、APIはかなり異なり、JSXベースではないため、ソリューションをアグノスティックかつタイプセーフにしているようです。現時点ではReactアダプターのみで、ドキュメントも不完全ですが、非常に有望です！

🔗 **参考リンク**:

* 🐦 [Tanner Linsley](https://twitter.com/tannerlinsley/status/1590101866385489929) がTwitterでプロジェクトを発表！素晴らしいトレーラー付きです！
* 🐦 [Ryan Florence](https://twitter.com/ryanflorence/status/1590194770936463362) は彼の仕事からインスピレーションを得てRemixの型安全性を向上させられるかもしれません。

[![Gatsby 5: The Fastest Gatsby Yet](/emails/issues/123/gatsby5.png)](https://www.gatsbyjs.com/blog/gatsby-5/)

**[Gatsby 5: 史上最速のGatsby](https://www.gatsbyjs.com/blog/gatsby-5/)**

Gatsby 5.0が正式リリースされました。レイアウト要素（ヘッダー、フッターなど）を変更する際のビルド時間を改善するGatsby Slice APIや、部分的なハイドレーションを可能にするServer Componentsのベータサポートが含まれています。また、新しいHead/Scriptコンポーネント、GraphiQL v2なども追加されています。

Gatsbyはまた、[Valhalla](https://www.gatsbyjs.com/blog/gatsby-5/)という新しいクラウドサービスを発表しました。これはGatsbyのソースプラグインを使用して、コンテンツハブと統一されたGraphQL APIを提供するもので、Gatsbyから切り離されています。[ValhallaはNext.jsでも使用可能](https://github.com/gatsby-inc/valhalla-examples/blob/main/frameworks/nextjs/app/\(pets\)/page.tsx)です。

静的サイトジェネレーターは現在あまり人気がありませんが、多くのサイトに適していると思います。少なくとも1つの主要プレイヤーがこの分野で革新を続けているのは良いことです。Slice APIは良いアイデアのように見えます。Valhallaがローカルで実行できればと思いますが、クラウドサービスのみのようです。

## 📱 React-Native {#react-native}

[![Expo SDK 47](/emails/issues/123/expo47.png)](https://blog.expo.dev/expo-sdk-47-a0f6f5c038af)

**[Expo SDK 47](https://blog.expo.dev/expo-sdk-47-a0f6f5c038af)**

新しいSDK 47が正式にリリースされました。新機能について詳しく解説した記事も公開されています。

* React Native 0.70.5とReact 18.1.0をサポート
* Expo Modules API 1.0により、Swift/Kotlinで簡単にFabricモジュールを作成可能
* Expo Go iOSでHermesが利用可能に、デバッグ体験が簡素化。次期SDKではHermesがデフォルトエンジンに
* 複数のExpoモジュールでFabricをサポート
* Expo Routerがベータ版に：React-Navigation向けのファイルシステムベースのルーティング
* EASの各種改善：エンドツーエンドセキュリティ、M1ワーカーなど

要約すると、非常に素晴らしいリリースです！Expo Modulesを使えば、Swiftをまだサポートしていないreact-native-codegenなどの公式ツールと比べて、新しいアーキテクチャの採用が容易になるようです。
