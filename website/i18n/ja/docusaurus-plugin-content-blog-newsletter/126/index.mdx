## 💸 Sponsor {#sponsor1}

**[FlyCode - コーディング不要でReactアプリを編集可能、Gitベース](https://bit.ly/3UPwAoJ)**

[FlyCode](https://bit.ly/3wWSi00)（YC S22）は、Reactウェブアプリを数分で編集可能にし、プロダクトチームとUXチームが迅速に反復・リリースできるようにします。開発者の時間を待つ（または消費する）必要がなくなります。

FlyCodeは通常のReactコードを読み取り、テキスト、画像、デザイントークン、分析イベントを特定します。その後、非エンジニアが共同で編集し、プルリクエストを送信できます（GitHubへのアクセス権は不要です）。

* 開発時間の節約
* コード統合不要、単にコードを読み取るだけ！
* GitHubベース、ボットがコードをスキャンしてプルリクエストを作成
* 既存のスタックとコンポーネントをそのまま使用可能
* コードベースの所有権を保持

こちらから始められます：[flycode.com/developers](https://www.flycode.com/developers)

## ⚛️ React {#react}

**[Reactはこの先どうなるのか？](https://www.infoxicator.com/is-react-going-anywhere)**

Rubenによる興味深い考察。あるe-commerceのCTOがReactを放棄しVanilla JSに移行した事例を受けて。アプリのパフォーマンス低下は本当にReactの責任か？代替技術の採用は正当化されるか？Reactは今後5年、10年にわたってリーダーであり続けるのか？

**[Reactのインタラクション時間を4倍改善](https://www.causal.app/blog/react-perf)**

Causalのクラウドスプレッドシートアプリにパフォーマンス問題が発生。Chromeプロファイラーを使った詳細な問題分析と解決策を紹介。AG Grid関連の問題（彼らがパッチを当てた）と、メモ化に関する自社コードの問題。Web Workersの使用も試みたが、シリアライズのコストが高すぎた。

* 📜 [RemixのuseFetcherフックをプログレッシブに強化する](https://sergiodxa.com/articles/progressively-enhance-the-usefetcher-hook-in-remix): JS無効ユーザー向けにRemixのuseFetcher APIをサポートする小技 - JS無効時はリダイレクトを強制。同様に、[ファイルアップロードのプログレッシブエンハンスメント](https://twitter.com/ebey_jacob/status/1596223682246234113)も可能。多くのアプリでは必須ではないが、必要時に実装可能なことを知っておくと良い。
* 📜 [Remix Deferredの概要](https://gist.github.com/jacob-ebey/9bde9546c1aafaa6bc8c242054b1be26): Remixの[defer](https://github.com/remix-run/remix/issues/4486)機能の解説。バックエンドからフロントエンドへPromiseを「転送」可能。Remixの内部動作を理解したい方向け。

* 📜 [ArConnectがChrome拡張をPlasmoフレームワークに移行](https://blog.plasmo.com/p/migrating-chrome-extension-to-plasmo): [Plasmo](https://github.com/PlasmoHQ/plasmo)使用レポート - Web拡張作成用のオープンソースReactフレームワーク。

**[React Nativeコアコントリビューターサミット2022](https://reactnative.dev/blog/2022/11/22/react-native-core-contributor-summit-2022)**

React-Nativeのコントリビューターの多くが9月初旬にReact-Native EUカンファレンスに参加しました。彼らは一連のワークショップに参加し、その報告を公開しています。新しいアーキテクチャ、Metroバンドラー、リリースワークフローなど、さまざまなトピックが議論されました。

**[Shopify - React Nativeのレンダリング時間計測ソリューション](https://shopify.engineering/measuring-react-native-rendering-times)**

[react-native-performance](https://github.com/Shopify/react-native-performance)パッケージの紹介。このパッケージは[フィールド/実ユーザーモニタリング](https://web.dev/lab-and-field-data-differences/)のパフォーマンスレポート（JSON形式）を生成できます。React-Navigationと簡単に統合可能で、これらのレポートを任意の分析ダッシュボードに送信できます。これにより、本番環境でパフォーマンス問題のある特定の画面を特定できます。これは既存のWebメトリクス（Core Web Vitals、TTI、INP）やVercel/Netlify/Gatsbyのパフォーマンスダッシュボードを彷彿とさせます...
