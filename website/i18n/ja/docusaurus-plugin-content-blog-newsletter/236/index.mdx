## ⚛️ React {#react}

[![Wake up Remix](/emails/issues/236/remix.jpg)](https://remix.run/blog/wake-up-remix)

**[Wake up Remix - Remix v3の計画](https://remix.run/blog/wake-up-remix)**

今週、[RemixがReactから離れる](https://x.com/AdamRackis/status/1925646909345058818)という噂を耳にしたかもしれません。RyanとMichaelは公式投稿でRemixの未来像を説明しました。

要約すると、React Routerは引き続き完全にメンテナンスされ、安全な選択肢です。並行して、Remix v3はPreactのフォークを基にした新しいフレームワークとして開発され、フルスタックをコントロールできるようになります。

Remix v3の目標は「ウェブフレームワークの可能性を再構築する」ことです。この投稿では、新しいフレームワークが従う指針（LLM最適化、Web API構築、コンポジションなど）が説明されています。

## 💸 Sponsor {#sponsor2}

[![Clerk Billingを使い始める](/emails/issues/236/clerk.jpg)](https://go.clerk.com/KfnV7Rj)

**[Clerk Billingを使い始める](https://go.clerk.com/KfnV7Rj)**

カスタムUIやWebhook、Stripeセッション管理なしで、Clerkを使って完全な課金エクスペリエンスを構築する方法を学びましょう。

* `<PricingTable />`コンポーネントで価格ページを作成
* Stripeと接続—手動のAPI呼び出しや同期は不要
* Clerkダッシュボード内でプランと権限を管理
* 顧客のプラン、機能、権限に基づいてアクセスを制御する`has()`ヘルパー
* ユーザーとサブスクリプションデータを一元管理

課金を有効にし、`<PricingTable/>`を`/pricing`ページに配置して[アプリケーションの収益化を開始](https://go.clerk.com/LphCouG)しましょう。
