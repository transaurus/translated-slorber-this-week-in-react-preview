## ⚛️ React {#react}

[![Next.js 15.2](/emails/issues/224/next.jpg)](https://nextjs.org/blog/next-15-2)

[**Next.js 15.2**](https://nextjs.org/blog/next-15-2)

VercelはNext.jsの新しいマイナーバージョンをリリースし、DXが大幅に改善されました。主なハイライト:

* エラーUIの再設計とスタックトレースの改善: Reactの新API[`captureOwnerStack()`](https://react.dev/reference/react/captureOwnerStack)を活用したデバッグ体験
* ストリーミングメタデータ: 非同期メタデータがページレンダリングやクライアントサイド遷移をブロックしなくなりました
* Turbopackのパフォーマンス改善: コンパイル時間の短縮とメモリ使用量の削減
* React View Transitions（実験的）: Reactの新しいView Transitions APIの実験的サポート
* Node.jsミドルウェア（実験的）: ミドルウェアでのNode.jsランタイム使用の実験的サポート

[![TanStack Form 1.0](/emails/issues/224/form.jpg)](https://tanstack.com/blog/announcing-tanstack-form-v1)

[**TanStack Form 1.0**](https://tanstack.com/blog/announcing-tanstack-form-v1)

長年の開発を経て、新しいTanStackライブラリが安定版としてリリースされました。発表記事では以下の特徴が紹介されています:

* フレームワーク非依存、ヘッドレス: 記事の例で使用されているReactを含む5つのフレームワークをサポート
* 極限の型安全性: `<form.Field>`というAPIは非常に優れています
* スキーマ検証: Zod、Valibot、ArkType、Standard Schemaをサポート
* 非同期検証、組み込みのデバウンス、AbortSignal
* アイソモーフィックでマルチプラットフォーム: クライアント/サーバー間で検証ロジックを共有可能

## 📱 React-Native {#react-native}

[![Lynx: ネイティブの可能性を解き放つ](/emails/issues/224/lynx.jpg)](https://lynxjs.org/blog/lynx-unlock-native-for-more)

[**Lynx: ネイティブの可能性を解き放つ**](https://lynxjs.org/blog/lynx-unlock-native-for-more)

ByteDanceが新たに発表したLynxは、ネイティブのクロスプラットフォーム技術です。TikTokを含む自社のプロダクションアプリで広く利用されており、Reactバインディングを備えているため、React Nativeの強力な競合となり得ます。他のフレームワークのサポートも予定されています。アーキテクチャの選択肢は興味深く、デュアルスレッドモデルと異なるランタイムを採用しています。React Nativeでさえまだサポートしていない一部のCSSやWeb APIをサポートしており、`<view>`のような便利な組み込みJSX要素を使用します。

ローンチ時の機能セットは印象的ですが、React 17 / Preactの使用や、ウェブ上でのカスタム要素など、好みが分かれる点もあるかもしれません。この技術が約束を果たし、既存のプレイヤーと真に競合できるかどうかは時間が証明するでしょう。いずれにせよ、競争は健全です。
