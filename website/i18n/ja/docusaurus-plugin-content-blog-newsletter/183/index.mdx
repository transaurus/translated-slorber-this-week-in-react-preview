## ⚛️ React {#react}

[![Vercel、エッジレンダリングをすべてNode.jsに戻す](/emails/issues/183/r183.jpg)](https://x.com/leeerob/status/1780705942734331983)

🐦 [Vercel、エッジレンダリングをすべてNode.jsに戻す](https://x.com/leeerob/status/1780705942734331983)

Vercelのプロダクト担当副社長であるLee Rob氏はXで、Vercelが「すべてのエッジレンダリングをNode.jsに戻す」ことを説明しました。Vercelを利用している方（おそらく利用していない方でも）は、Vercelが長い間「エッジ」という概念を推進してきたことをご存知でしょう。一方で、エッジとは、Vercelでホストされているサイトにアクセスするユーザーの近くでコンピュートが実行されることを意味します。もう一方で、このコンピュートはNode.jsではなくV8上で構築された「エッジランタイム」で実行されることも意味します。エンドユーザーの近くでコンピュートを実行するという概念は魅力的に聞こえますが、データを中央の場所からロードする必要がある場合（これは非常に頻繁に起こります）、その利点はすぐに消えてしまいます。同時に、開発チームはエッジランタイムで利用可能なAPIの限られたサブセットにワークロードを適応させるのに苦労することがよくあります。現在、Vercelは開発者にとっての課題が限られた利点を上回るとの結論に達し、集中型のNode.jsランタイムに戻すことを決めたようです。この透明性には感謝します！詳細を知りたい方は、Theoが[動画](https://youtu.be/lage-k1Zfrg?si=5SP4hyoFkixTsh-y)で解説しています。

* **数分でAIチャットインターフェースを構築** - 高品質な会話型AIインターフェースを数行のコードで実現
* **Reactコンポーネント & フック** - UI用の`<AiChat />`と簡単な統合のための`useChatAdapter`フック
* **柔軟なアダプター** - `ChatGPT`、`LangChain`、`HuggingFace`に対応、あらゆるバックエンド用に**独自アダプター作成**可能なAPI
* **UI状態とAIバックエンドを最適に同期**
* **マークダウンストリーミング** - 生成中のLLMマークダウンをレンダリング
* **ゼロ依存** - 軽量なコードベース

詳細情報、例、ドキュメントは[nlux.dev](https://nlux.dev?c-twr2-p1)でご覧ください

📦 [React Native 0.74リリース](https://reactnative.dev/blog/2024/04/22/release-0.74)

今週、React Native 0.74がリリースされ、コミュニティにとって大きなマイルストーンとなりました！最新バージョンでは、New Architectureを使用する新規プロジェクトにおいて「ブリッジレスモード」がデフォルトになりました（次のリリース0.75ではNew Archもデフォルトになります）。どちらも必要に応じて無効にできますが、これらは明らかにReact Nativeの未来を表しており、特にここ数ヶ月で多くのライブラリが両方のサポートを追加しています。それだけではありません：RN 0.74にはYoga 3.0が含まれています。破壊的変更や非推奨はほとんどありませんが、[リリースノート](https://reactnative.dev/blog/2024/04/22/release-0.74)で確認する価値があります。
