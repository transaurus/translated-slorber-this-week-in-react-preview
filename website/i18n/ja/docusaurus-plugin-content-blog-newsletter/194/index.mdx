* 👀 [Rsnext](https://github.com/ScriptedAlchemy/rsnext/commits/rspack): RspackチームはNext.jsをRustバンドラーで動作させる取り組みを進めており、大きな進展を共有しています。最初のアルファ版が公開され、[「next」を「rsnext」にエイリアスすることで試用可能](https://github.com/substancelabs/rsnext/blob/main/package.json#L11)。また、[Rspack 1.0 beta 1が「layers」機能を搭載してリリース](https://x.com/rspack_dev/status/1818490358319530221)。この機能はアプリを複数のバリアントでバンドルし、React Server Componentsをサポートするのに有用です。本番環境対応が整い次第、さらに詳しく紹介する予定です。
* 👀 [Flutter WebプロジェクトにおけるNext.js統合によるSEO強化](https://gist.github.com/lcw99/ac1738cb6d5bb032085077605f56665e): Googleが[HTMLレンダラーの廃止と削除を表明](https://github.com/flutter/flutter/issues/145954)した後、FlutterユーザーはWeb SEO問題の解決に躍起になっています。このやや不自然な回避策はNext.jsを基にしています。

* 📜 [React Owner Components](https://reacttraining.com/blog/react-owner-components): OwnerとParentコンポーネントの違いを非常に分かりやすく説明。OwnerはJSXを作成し、それにpropsを渡します。Client ComponentsはServer Componentsを所有できません。
* 📜 [Partial Pre Renderingの解剖](https://www.nikhilsnayak.dev/blogs/dissecting-partial-pre-rendering): Next.jsの実験的PPR機能が他のレンダリング戦略の長所をどのように取り入れているかを具体的な例（動的レビュー付き静的コースページ）で解説。
* 📜 [Vercel - Next.jsにおけるFlags as Code](https://vercel.com/blog/flags-as-code-in-next-js): 新しいパッケージ`@vercel/flags/next`がNext.jsでフィーチャーフラグを統合するベストプラクティスをどのようにエンコードするかを説明。基盤となるフラグプロバイダーソースからコードを分離します。フラグは常にサーバー側で評価されるため、レイアウトシフト、スピナー、クライアントへの機密情報開示を防止。静的ページやPPRでも動作する点が非常に優れており、フラグの組み合わせ分だけ静的バリアントを事前計算可能！
* 📜 [Netlify - GatsbyのLive Previewが新しいDev Serverで復活（そして進化）](https://www.netlify.com/blog/gatsby-live-preview-with-new-dev-server/): NetlifyはフレームワークのDev Serverをクラウドで起動する新しい汎用プラットフォームプリミティブを提供。互換性のあるコンテンツ（CMS）ソースプラグインを使用する際のGatsbyライブプレビューに有用。
* 📜 [大規模SPAにおける柔軟なネットワークデータプリローディング](https://mmazzarolo.com/blog/2024-07-29-data-preloading-script/): フレームワークを使用しない場合、HTMLにデータプリローディングロジックをインライン化することで、データ/バンドルフェッチの並列化によりアプリ起動時間を短縮可能。
* 📜 [独自ストアの作成](https://www.smallrecipesfordisaster.com/posts/local-store): `useSyncExternalStore`でReactに接続する独自ストアを作成するために必要な要素を示す。
