## React

[**Statements Vs. Expressions**](https://www.joshwcomeau.com/javascript/statements-vs-expressions/?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Josh Comeauは新しいReactコースの準備中です。彼はJavaScriptの基本に戻り、JSXがどのように動作するかをより深く理解することを提案しています。JSXは式でのみ動作します。同じトピックで、[do-expressions](https://github.com/tc39/proposal-do-expressions?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)の提案が進むことを願っています😇。

[**How to build connected components in Storybook**](https://storybook.js.org/blog/how-to-build-connected-components-in-storybook/?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Storybookの公式記事で、デコレーターを使ってできること（レイアウトの追加、グローバルプロバイダー、Reduxストアやリクエストのモックなど）がよく説明されています。コンテキスト依存のためにレンダリングできない「接続された」コンポーネントをStorybookに追加するのに役立ちます。

[**npmパッケージの市場シェアをどう推定しているか（Reduxの使用率が他のライブラリと比べてどうか）**](https://blog.isquaredsoftware.com/2022/07/npm-package-market-share-estimates/?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

Mark Erikson（Redux）が、さまざまな信頼性の異なるツールに基づいてnpmパッケージの使用トレンドを分析する方法を説明しています。ダウンロード統計はCIビルドに関連することが多いです。彼はReactの状態管理ライブラリを例に挙げています。現在、Redux/Reactの使用率は33％と推定されています（以前は45-50％でした）。

[**Reactにおけるアクションレスでステートレスなリデューサー**](https://julesblom.com/writing/actionless-and-stateless-reducers?utm_campaign=This%20Week%20In%20React\&utm_medium=email\&utm_source=Revue%20newsletter)

useReducer()は複雑なケースだけに限定されません。falseからtrueにブール値を変更するような非常にシンプルなステートマシンにも使用できます。
