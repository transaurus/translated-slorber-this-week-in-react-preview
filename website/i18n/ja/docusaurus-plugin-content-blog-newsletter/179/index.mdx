## ⚛️ React {#react}

[![RedwoodJSでReact Server Componentsが利用可能に](/emails/issues/179/redwood.jpg)](https://redwoodjs.com/blog/rsc-now-in-redwoodjs)

[**RedwoodJSでReact Server Componentsが利用可能に**](https://redwoodjs.com/blog/rsc-now-in-redwoodjs)

v8.0から始まるRedwoodの次の「時代」であるBighornは、React Server Componentsに全面賭けします。この記事では、なぜRedwoodがGraphQLよりもRSCを優先するようになったのか、そしてGraphQLがほとんどの開発者やプロジェクトにとって最適なDXではなかった理由を説明しています。GraphQLは引き続きサポートされ、RSCと同じアプリ内で併用できます。Redwood Cellsでは、新しい「data」関数をエクスポートしてCellデータをロードできるようになりました。

Redwood Bighornはv8のカナリーリリースとして今日から試すことができますが、ブログ記事で言及されているように、まだ多くの未完成部分があり、正式なメジャーリリース前に修正される予定です。また、[Redwoodのルーティング規則をNext.jsやRemixと比較する](https://redwoodjs.com/blog/whats-different-comparing-the-router-in-nextjs-app-api-nextjs-pages-api-remix-and-redwoodjs)興味深い記事も読む価値があります。

このセクションは現在[Benedikt](https://twitter.com/bndkt)と共同執筆しています。コメントはメールまたは[Twitter](https://twitter.com/bndkt)でお送りください！

[![React Navigation - 静的APIの紹介](/emails/issues/179/rnav.jpg)](https://reactnavigation.org/blog/2024/03/25/introducing-static-api)

[**React Navigation - 静的APIの紹介**](https://reactnavigation.org/blog/2024/03/25/introducing-static-api)

React Nativeで最も使用され、最も重要なライブラリの1つであるReact Navigation。現在のメジャーバージョン（6）は約3年前に導入されましたが、次のメジャーリリースが近づいているようです。v7（現在アルファ版）では、React Navigationは静的APIを（再）導入し、オブジェクト設定から画面を定義できるようになります。これにより型推論が可能になり（動的なReact Navigationツリーの型を作成したことがある方ならその負担が軽減されることをご存知でしょう）、ディープリンクの設定も容易に（あるいは部分的に自動化さえ）なります。

他の変更点について知りたい方は、[v7アップグレードガイド](https://reactnavigation.org/docs/7.x/upgrading-from-6.x/)も必ずお読みください！
